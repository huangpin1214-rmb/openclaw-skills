# Fabless 公司 PCN 发布 Checklist

## 适用范围
Fabless 公司向客户发布 PCN（Product Change Notification）时的标准化流程。

---

## 一、触发阶段

- [ ] 识别 PCN 触发条件（设计变更、代工厂变更、封装变更、规格调整、停产/EOL）
- [ ] 确认变更是否影响form-fit-function（需评估是否需要客户承认）
- [ ] 确认变更是否涉及 AEC-Q 认证范围（车规产品需额外流程）
- [ ] 确定 PCN 类型：**重大变更** 还是 **轻微变更**
- [ ] 指定 PCN 负责人（Project Owner）

---

## 二、内部评审

- [ ] 变更技术方案评审完成
- [ ] 代工厂/封装厂变更确认书获取
- [ ] 内部质量（QA）评审通过
- [ ] 法规合规评估（出口管制、REACH、RoHS 等）
- [ ] 确定变更生效日期（Effective Date）

---

## 三、文档准备

- [ ] PCN 表格填写完成，包含：
  - [ ] PCN 编号（内部编号 + 客户编号映射）
  - [ ] 变更产品型号/系列（含旧版和新版对比）
  - [ ] 变更类型（设计/代工/封装/规格/停产）
  - [ ] 变更描述（文字 + 图表）
  - [ ] 变更原因
  - [ ] 变更影响评估（电气参数、热参数、机械尺寸）
  - [ ] 替代方案（如果有）
  - [ ] 生效日期
  - [ ] 样品提供计划
  - [ ] 联系人信息
- [ ] 内部审批流程完成（签核记录）

---

## 四、样品与验证

- [ ] 确认是否需要提供**变更对比样品**（Change Sample vs. Production Sample）
- [ ] 确定样品数量和发放时间节点
- [ ] 编制**变更验证报告**（Change Qualification Report）
- [ ] 可靠性测试报告（AEC-Q100/Q101/Q200 if applicable）
- [ ] 电性参数对比报告（ Bench characterization data）
- [ ] 确认客户提供 **Acknowledgment** 的时间窗口（通常 30 天）

---

## 五、客户通知

- [ ] 确认客户清单（直接客户 vs. 分销商渠道）
- [ ] 通过正式渠道发送 PCN（邮件/Portal/系统）
- [ ] 通过客户承认请求（PCN Acknowledgment Request）
- [ ] 记录每位客户的确认状态
- [ ] 处理客户问询和异议
- [ ] 确认 Last-Time-Buy（LTB）窗口（如适用）

---

## 六、变更实施

- [ ] 跟踪客户 Acknowledgment 回收率（目标 100%）
- [ ] 未回复客户进行二次跟进
- [ ] 内部变更启动会议
- [ ] 新版产品切换执行
- [ ] 旧版产品退市（EOL）执行（如适用）
- [ ] 库存消化计划确认
- [ ] 记录所有客户承认文件

---

## 七、存档与回顾

- [ ] PCN 全流程文档存档（电子档 + 纸质签核）
- [ ] 更新产品变更履历表
- [ ] PCN 有效性回顾（变更是否按计划执行）
- [ ] 客户反馈汇总分析
- [ ] 经验教训记录（Lessons Learned）

---

## 附：PCN 类型快速判断

| 变更类型 | 是否需要客户承认 | 样品需求 | LTB需求 |
|---------|--------------|--------|--------|
| 封装变更 | ✅ 通常需要 | ✅ | ✅ |
| 代工厂变更 | ✅ 通常需要 | ✅ | ✅ |
| 规格收紧 | ⚠️ 评估影响 | 可能需要 | ❌ |
| 规格放宽 | ⚠️ 评估影响 | 可能需要 | ❌ |
| 标识变更 | ❌ 不需要 | ❌ | ❌ |
| 停产（EOL） | ✅ | ❌ | ✅ |

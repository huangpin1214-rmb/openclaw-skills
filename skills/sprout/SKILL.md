# Sprout - 内容深度分析与洞察生成

对任意输入内容进行多维度深度分析，提炼核心观点，关联背景知识，生成有洞察力的解读报告。

## 触发词

- "分析一下"
- "发芽"
- "深度解读"
- "给点洞察"
- "解读这段话"
- "读取我的笔记后发芽"

## 核心能力

1. **笔记关联** — 搜索 Get 笔记中相关内容，融入解读
2. **种子提取** — 从内容中提取1-3个核心洞察点作为"种子"
3. **多维解读** — 从8个知识维度深度延伸
4. **素材沉淀** — 支持输出后存入Get笔记

## 分析维度（8维解读）

每个种子必须从以下维度中选3-5个最相关的进行解读，不要每次都用同样的维度：

| 维度 | 思考方向 |
|------|---------|
| **🔬 科学原理** | 物理学/生物学/心理学等底层规律，类比到现象 |
| **📜 历史脉络** | 类似事件在历史上如何发生、演变、结局 |
| **🏢 商业逻辑** | 商业模式、竞争策略、价值链、盈利逻辑 |
| **⚡ 第一性原理** | 追问"为什么"，追溯到最根本的因果链 |
| **🧠 认知心理** | 人类认知偏差、思维定式、决策盲区 |
| **🔮 反脆弱** | 在压力/混乱中反而变强的案例和规律 |
| **🌐 系统思维** | 整体vs局部、反馈回路、延迟效应 |
| **💡 创新范式** | 类似创新如何发生、扩散、取代旧范式 |

**选择维度的原则**：
- 选与内容最相关的3-5个，不要面面俱到
- 技术类内容优先：第一性原理、科学原理、商业逻辑
- 管理/人生类优先：历史脉络、认知心理、反脆弱
- 热点事件优先：系统思维、创新范式、商业逻辑

## 工作流程

### Step 1: 笔记关联（如适用）

当用户说"读取我的笔记发芽"或包含相关意图时：
1. 提取输入内容中的**2-3个关键词**
2. 调用 `scripts/fetch_notes.py` 搜索 Get 笔记：
   ```bash
   python3 skills/sprout/scripts/fetch_notes.py "<关键词1>" 3
   ```
3. 获取最相关的笔记片段（最多3条）
4. 将笔记内容融入后续解读环节
5. 在结尾注明"结合你的[笔记标题/主题]解读"

### Step 2: 提取种子

从输入内容中提炼1-3个核心洞察点作为种子，每颗种子用1-2句话概括。

### Step 3: 选择维度

根据内容类型选择最相关的3-5个维度。

### Step 4: 生成解读

每个种子从选定的维度展开分析，要有具体的案例、数字或类比。

### Step 5: 提炼金句

每颗种子产出一句 Aha Moment，不超过20字。

### Step 6: 素材沉淀

输出完成后询问：
> "需要把这篇发芽存到Get笔记吗？"

如用户确认，调用 Get 笔记 API 存入，标签：`#发芽 #洞察 #素材`

## 输出格式

```
## 🌱 [主题名称]

> **种子**
> [核心洞察，不超过2句话]

> **📖 关联笔记**
> [当有笔记关联时显示：相关笔记片段]

### 解读
[从3-5个维度深度展开，每个维度1-3段，要有具体的案例或类比]

**✨ Aha Moment**
> "[一句点睛之笔，20字以内]"

---
（如有多颗种子，重复以上结构）
```

## 配套脚本

| 脚本 | 用途 |
|------|------|
| `scripts/fetch_notes.py` | 搜索 Get 笔记相关内容 |

### 直接调用示例

```bash
# 搜索 Get 笔记
python3 skills/sprout/scripts/fetch_notes.py "私有知识 AI" 3

# 返回格式
# {
#   "note_id": "1906078867177610808",
#   "title": "...",
#   "content": "笔记片段...",
#   "created_at": "2026-04-03"
# }
```

## 注意事项

- 解读要有具体的**案例、人物、数字**，不要只讲空洞的道理
- Aha Moment 要**短促有力**，不超过20字
- 不要过度分析，3-5个维度足够
- 保持洞察的**原创性**，不要堆砌常见鸡汤
- **优先关联笔记**：当用户提到"我的笔记"或历史背景时，必须先搜索笔记再发芽
